Every color in Vespera is a CSS variable on `.vsp-root` — components never hardcode a hex.
That's what makes the whole system re-theme with one attribute. The swatches below render the
**live dark-theme** tokens; switch `data-theme="light"` and every one of them swaps. To change
them, see [Theming](/vespera-ui/guides/theming/).

## Accent

The brand/primary color. `--accent` drives buttons, links, focus rings, and the background glow;
`--accent-2` is the secondary used in gradients and charts. `--accent-ink` is the text color that
sits **on** an accent fill.

<SwatchGrid>
  <Swatch name="accent" value="#4a7cff" note="Primary / brand" />
  <Swatch name="accent-2" value="#b16cff" note="Secondary / gradient" />
  <Swatch name="accent-ink" value="#ffffff" note="Text on accent" />
</SwatchGrid>

Set the accent inline — no rebuild:

```html
<div class="vsp-root" style="--accent: #1fb574">…</div>
```

## Surfaces

Vespera builds depth with a three-step surface scale over the page background, not with heavy
shadows. `--bg` is the page; `--surface-1/2/3` are progressively raised. See
[Surfaces & background](/vespera-ui/foundations/surfaces/) for how they stack.

<SwatchGrid>
  <Swatch name="bg" value="#080b14" note="Page background" />
  <Swatch name="surface-1" value="#0e131f" note="Cards, panels" />
  <Swatch name="surface-2" value="#141a28" note="Inputs, raised rows" />
  <Swatch name="surface-3" value="#1a2233" note="Popovers, hover wells" />
</SwatchGrid>

## Text

A three-level hierarchy — body, dimmed, and faint — so you set emphasis with a token instead of
opacity.

## Semantic

Status colors for badges, alerts, toasts, and validation. Use these — never raw greens/reds — so
they adapt across themes.

<SwatchGrid>
  <Swatch name="success" value="#34d399" note="Positive / done" />
  <Swatch name="warning" value="#fbbf24" note="Caution / pending" />
  <Swatch name="danger" value="#fb7185" note="Error / destructive" />
  <Swatch name="accent" value="#4a7cff" note="Info / neutral-active" />
</SwatchGrid>

:::tip[Mixing colors]
Need a tint? Use `color-mix` with a token instead of a new variable — e.g. a 14%-accent chip
background: `background: color-mix(in oklab, var(--accent) 14%, transparent)`. That's exactly how
the components build their subtle fills, so your additions stay theme-aware.
:::
